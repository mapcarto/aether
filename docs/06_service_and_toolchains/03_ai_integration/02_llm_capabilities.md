# 接入后大模型能做什么

软件齐备后，大模型通过任意一个接入的 CLI，可以完成四类任务：感知、分析、执行、表达。所有能力均通过 Gateway 暴露的工具发起，不由模型直接消费 UDP 协议或连接渲染引擎。

---

## 1. 四类任务

### 1.1 感知

访问 AeServer 的当前状态，获取用于决策的基础信息。

感知数据按三层返回，模型从宏观到微观渐进聚焦：

| 层级 | 数据范围 | 返回形式 | 用途 |
|---|---|---|---|
| **宏观层（Macro）** | 金字塔 LOD 0-2 | 热点热力摘要、区域密度概览 | 快速建立全局态势，极低 Token 消耗 |
| **焦点层（Focus）** | 局部扇区（少量实体） | 结构化实体摘要（速度、航向、标签） | 局部问题分析与推理 |
| **证据层（Evidence）** | 最小证据集（冲突预测、状态跃迁） | 结构化摘要或 View Token | 执行前验证，避免几何幻觉 |

当查询命中海量体素（如整片建筑群的精密网格）时，Gateway 强制截断文本序列化，返回 View Token 句柄。模型持有句柄，将其透传给渲染客户端或 `Visual_Snapshot_Bake` 工具，不在上下文中处理原始数据。

所有查询必须附带预算约束（最大 LOD 深度、最大实体数、最大时间窗口、最大 Token 预算），超限则拒绝全量返回，改为降级或句柄模式。

典型操作：

- 查询指定区域的实体分布与密度热度。
- 获取某个实体的当前状态（位置、速度、标签）。
- 拉取最近一段时间的事件列表。

### 1.2 分析

基于感知结果，调用分析服务完成高阶空间推理。**所有空间几何计算必须卸载至分析工具，禁止模型自行推导。**

典型操作：

- 碰撞风险判断：调用 `Spatial_Boolean_Check`，由底层 C 算法库完成布尔碰撞计算。
- 轨迹预测：调用 `Trajectory_Predict`，基于历史状态推算未来位置。
- 约束冲突识别：调用 `Constraint_Compliance_Check`，检查操作是否违反空域规则或业务约束。

返回形式：结构化分析结论，供模型据此生成执行提案或向用户说明。

### 1.3 执行

提出状态变更提案，经 AeServer 校验通过后提交。执行类操作必须走 Propose→Commit 链路，不能绕过校验直接提交。

典型操作：

- 提出实体状态变更（位置调整、航向修正）。
- 冻结空域或划定临时限制区。
- 控制实体执行指定动作。

流程：

1. 模型生成提案，CLI 执行权限 hooks。
2. Gateway 把提案发送至 AeServer 沙盒校验，返回 `transaction_id`。
3. CLI 持有合法 `transaction_id` 后提交 Commit，AeServer 执行并写入审计链。

### 1.4 表达

触发渲染或生成报告，输出供人类或上层系统消费的结果。

典型操作：

- 调用 `Visual_Snapshot_Bake` 生成当前状态的可视快照（Base64 JPEG）。
- 输出任务完成情况的自然语言说明。
- 生成处置建议或决策报告。

返回形式：快照句柄、图片 URL 或结构化文本，由 CLI 展示或转发给用户。

## 2. 本地处理与服务端处理的分工

在实际接入中，确实存在把数据拉到本地处理的需求，例如 Gemini + 本地交互 Viewer 这类场景。本质上这类场景是"本地交互壳 + 服务端能力"——前端在本地做视图交互，真正的查询、计算和执行仍调用后端服务。

### 2.1 分工原则

- **适合本地处理**：小数据量、短时间窗、以交互展示为主的任务。例如当前视窗内的实体渲染、局部筛选、临时标注。
- **必须在服务端处理**：大数据量、需要全局一致性的计算，以及所有会改写状态的操作。
- **执行类操作无例外**：无论使用哪种 CLI，提案和提交都必须经过 Gateway + AeServer 完成，本地 CLI 不能绕过。

### 2.2 默认数据策略

1. Gateway 默认返回摘要或句柄，不做全量数据推送。
2. 本地只缓存当前视窗或当前任务所需的数据切片。
3. 超过带宽或体量阈值时，强制切换为服务端计算 + 句柄回传。

### 2.3 典型场景对照

| 场景 | 数据在哪里处理 | 工具运行位置 |
|---|---|---|
| 当前视窗实体可视化 | 本地（拉取切片后渲染） | 本地 Viewer |
| 区域碰撞风险分析 | 服务端（分析服务） | 服务端 MCP tool |
| 区域全量数据导出 | 服务端（分批次返回句柄） | 服务端 MCP tool |
| 实体状态修改 | 服务端（Gateway + AeServer） | 服务端 Propose/Commit |
| 快照渲染 | 服务端（渲染服务） | 服务端 MCP tool，本地展示句柄结果 |
